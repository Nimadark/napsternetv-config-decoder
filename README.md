# White-Box AES Configuration Decoder for NapsternetV

This project is a high-performance, standalone PHP implementation engineered specifically to parse, validate, and decrypt custom encrypted configuration profiles (such as `.npvt` files) generated by or used in the **NapsternetV** mobile application.

---

## What is NapsternetV?
NapsternetV is a popular v2ray/Xray-core client tool for Android and iOS that allows users to connect to private VPN protocols (including VLESS, VMess, Trojan, and Shadowsocks). To protect backend server architecture, UUIDs, and network keys from unauthorized access, the app exports its configuration payloads using a custom encrypted format secured with **White-Box Cryptography**.

---

## Core Functionality of this Decoder
When NapsternetV exports a configuration string, it prefixes it with a structural header (`NPVT1`) followed by three distinct, comma-separated Base64 sections. This PHP engine acts as the definitive server-side counter-part to decode that data by executing the following technical pipeline:

### 1. Header & Signature Validation
The script inspects the incoming raw text stream, automatically cleans up potential copy-paste artifacts (such as spaces, tabs, or trailing `\n` carriage returns), and enforces strict magic-byte validation to ensure the configuration belongs to the `NPVT1` protocol.

### 2. White-Box AES CTR Blocks Evaluation
Instead of using standard, vulnerable cryptographic keys that could easily be extracted via static analysis, the application utilizes **White-Box AES Cryptography in Counter (CTR) Mode**. This approach embeds the keys directly inside vast, multi-dimensional mathematical lookup tables (LUTs) to resist reverse engineering. 

This PHP engine accurately mirrors the client's internal cryptographic behaviors, handling:
* **Unsigned 32-bit shifting (`>>>`)** logic natively in PHP.
* Matrix row mutations via a custom **`ShiftRows`** implementation.
* Systematic Big-Endian byte evaluations to maintain exact matching with Kotlin/Java counter increments.

### 3. Multi-Round Table Mapping & Optimization
The algorithm processes the encrypted binary streams through layered table structures extracted directly from the app's memory (specifically mapping properties like `xor`, `tyboxes`, `mbl`, and `tboxesLast`). To achieve near-zero execution latency on a web server, the engine includes an asset conversion utility that translates massive raw JSON dumps into native, memory-cached PHP serialized arrays.

### 4. Payload Extraction
Once the cryptographic barrier is removed, the engine automatically decrypts the structure into human-readable text. It verifies internal configuration version constraints and parses the remaining structural segments into clean, natively accessible PHP associative arrays:
* **`servers`**: The plaintext list containing decrypted proxy types, backend server IPs, connection ports, encryption parameters, and security tokens.
* **`setting`**: The primary operational flags, core routing matrices, and client-side system profiles.

---

## Primary Use Case
This engine is designed for automation systems, API management panels, and user control dashboards that need to dynamically decode, authenticate, or manage shared access links generated from NapsternetV profiles without relying on client-side environments or running an emulator.

---

## How to Use

### 1. Prepare Your Lookup Tables
Ensure your multi-dimensional mathematical lookup tables extracted via Frida are saved as `perfect_dumped_tables.json` in the root directory of your project.

### 2. Optimize Your Tables for Production (Recommended)
Because full White-Box AES lookup tables are extremely massive, parsing the raw JSON file on every incoming API request causes intensive CPU and memory overhead. It is highly recommended to run the optimization script once to generate a native, memory-cached PHP static array:

Create a file named `convert.php` and execute it once:
```php
<?php
// convert.php
$data = json_decode(file_get_contents('perfect_dumped_tables.json'), true);
file_put_contents('tables_optimized.php', "<?php\nreturn " . var_export($data, true) . ";\n");
